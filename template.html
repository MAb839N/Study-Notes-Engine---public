⚙ Maintenance Guide

Rules, workflow, and templates for expanding this site — for you or any AI assistant.

🗂

Project Structure

This is a build-based static site. You never edit study-notes.html directly. The workflow is:

project/
├── build.py          ← run this to rebuild
├── template.html     ← shell: CSS + JS only. Rarely changed.
├── modules/
│   ├── pom-t1.md     ← one file per module (AI-generated)
│   ├── rop-w1.md
│   └── ...
├── AI_MODULE_GUIDE.md  ← rules for AI generating new modules
└── study-notes.html  ← FINAL OUTPUT. Share/sync this file.

To add a new module: create a .md file in modules/, run python build.py. That is the entire workflow.

To rebuild: python build.py (Windows) or python3 build.py (macOS/Linux). Zero install required.

📜

Established Rules & Requirements

These rules govern all content. Preserve them in all AI prompts and future edits:

  • R1No institution names or course codes. Use course titles and short forms only (e.g. "Principles of Management (POM)").
  • R2Mindmap required per topic. Every module must include an interactive SVG mindmap. Each node links to its section via onclick="goSec('section-id','page-id')".
  • R3Zero external dependencies. No CDN fonts, no remote scripts. The output HTML must work fully offline in any major browser.
  • R4This maintenance page must always exist and be consulted before any modification.
  • R5All facts must be exact and correct. Do not paraphrase definitions loosely. Accuracy matters for study use.
  • R6Expandable by design. New modules are added via the build pipeline — never by hand-editing study-notes.html.
  • R7Three study modes per topic: Notes (collapsible sections), Mindmap (interactive SVG), Flashcards (flip cards). All three required.
  • R8Consistent colour coding. blue=definitions, green=functions/processes, gold=steps/examples, rose=warnings/skills, purple=structure, teal=rationale/why, orange=types, slate=summary.
  • R9System fonts only. Font stack: --font-d (Palatino/Georgia), --font-b (Segoe UI/system-ui), --font-m (Consolas/Courier New).

Module File Format (modules/*.md)

Each module file has three parts separated by markers. See AI_MODULE_GUIDE.md for the full schema — always pass that file to the AI when generating new content.

---
id: course-label            ← e.g. pom-t1, rop-w2 (lowercase, no spaces)
course: POM                 ← short course code
course-full: Principles of Management
label: Lesson 1             ← shown in sidebar and breadcrumb
title: Introduction to...   ← shown in hero and home card
description: One sentence summary.
reference: "Author (Year). Title. Publisher."  ← optional
pills:
  - Section Title | section-id    ← quick-jump buttons in hero
  - Another Section | another-id
---

<!-- NOTES -->
<section id="sec-id" icon="📌" color="blue" title="Section Title" num="1">
  <!-- Section body HTML here -->
</section>

<!-- MINDMAP -->
<svg viewBox="0 0 1000 500" ...> ... </svg>

<!-- FLASHCARDS -->
<div class="fc-sec"> ... </div>

Section color values: blue · green · gold · rose · purple · teal · orange · slate

Critical: Never use </section> inside section content — it will break the build parser.

Component Templates

Definition box

<div class="def-box">
  <div class="def-lbl">Core Definition</div>
  <p><strong>Term</strong> is the definition.</p>
</div>

2-column / 3-column grid

<div class="g2">  <!-- or g3 for 3 columns -->
  <div class="card-g cg-plan">
    <h4><span class="f-badge bg-gr">P</span><span class="clr-gr">Planning</span></h4>
    <p>Card content.</p>
  </div>
</div>

Flip flashcard

<div class="flip-card" onclick="this.classList.toggle('flipped')">
  <div class="flip-in">
    <div class="fc-f">
      <div class="fc-lbl">Label</div>
      <div class="fc-q">Question?</div>
      <div class="fc-tip">Tap to reveal ↗</div>
    </div>
    <div class="fc-b">
      <div class="fc-a">Answer with <strong>key terms</strong>.</div>
    </div>
  </div>
</div>

Mindmap SVG node + branch

<!-- Branch path -->
<path d="M fromX fromY C cp1X cp1Y, cp2X cp2Y, toX toY"
      fill="none" stroke="#COLOR" stroke-width="4" stroke-linecap="round"/>

<!-- Clickable label node -->
<g class="mm-node" onclick="goSec('section-id','page-id')"
   onmouseenter="showTip('Heading','Tooltip text')" onmouseleave="hideTip()">
  <rect x="X" y="Y" width="W" height="H" rx="4" fill="rgba(R,G,B,.09)"/>
  <text x="TX" y="TY" text-anchor="start" fill="#COLOR"
        font-family="Arial,sans-serif" font-size="12.5" font-weight="700">Label</text>
</g>

SVG viewBox: 0 0 1000 500. Centre node: ~(490, 250). Left nodes: text-anchor="end". Right nodes: text-anchor="start". Main branches stroke-width="4", sub-branches stroke-width="2".

Quick Tips

  • Sidebar search filters visible .sec blocks by text and auto-expands matches. It only searches the currently active page.
  • When drawing a new mindmap: sketch on paper first. ViewBox is 0 0 1000 500. Centre node near (490, 250). Use cubic Bezier paths (C) for smooth curves.
  • Flashcard flip is CSS-only — use the flip-card / flip-in / fc-f / fc-b structure. No JS needed per card.
  • The goSec(sectionId, pageId) function: switches to Notes mode, then scrolls to and flashes the target section. Always pass both arguments.
  • Module files are sorted alphabetically from the modules/ folder. Name files consistently: pom-t1.md, pom-t2.md, rop-w1.md.
